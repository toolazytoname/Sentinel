## 1. 执行底座对比（六大框架）

| | freqtrade | jesse | nautilus_trader | OctoBot | hummingbot | vnpy |
|---|---|---|---|---|---|---|
| Star | 52.1k | 8.1k | 24.4k | 6.2k | 19.1k | 42.7k |
| 许可证 | GPL-3.0 | MIT（live 闭源） | LGPL-3.0 | GPL-3.0 | Apache-2.0 | MIT |
| 2026 活跃度 | 极活跃（月度发版） | 活跃 | 极活跃 | 放缓 | 活跃 | 偏慢 |

| Telegram 控制 | ✅ 内置 | 仅通知 | ❌ | ✅ | 独立组件 | ❌ |
| 引擎级回撤熔断 | ✅ **唯一开箱即用**（MaxDrawdown Protection） | ❌ 需自写 | 可自建 | ❌ | kill switch | ❌ |
| 交易所侧止损 | ✅ | ❌ | ✅ | ❌ | ❌ | N/A |
| 回测→dry-run→实盘 | 一体，一个配置切换 | 一体（live 闭源） | 一体 | 一体 | 回测弱 | 非加密 |
| 未来函数检测 | ✅ lookahead-analysis | 设计上防 | ✅ | ❌ | ❌ | ❌ |
| 加密交易所 | 10+ CEX + Hyperliquid | 8+ | 12+ | 15+ | 最广（含DEX） | **核心已移除** |

**结论：freqtrade 为执行底座。** 理由：
1. 唯一内置引擎级最大回撤熔断（`MaxDrawdown`/`StoplossGuard`/`CooldownPeriod` Protections）——稳健型需求的核心
2. 交易所侧止损单：进程崩溃后仓位仍有保护——7x24 无人值守的底线保障
3. FreqUI + Telegram 双通道远程监控
4. dry-run→实盘一个配置项切换，天然支持"回测→模拟→小额→大额"流程
5. 社区最大、十年月度发版、Docker 部署成熟

**备选定位**：
- jesse：研究台（Monte Carlo、bootstrap 显著性检验是六者最强），但 live 插件闭源绑定厂商账号，不用于实盘
- nautilus_trader：未来若走 tick 级/多场所再迁移，第一版不用（无 UI，自建监控投入以月计）
- vnpy：排除（核心仓库已无加密交易所网关，Qt 桌面形态不适合云端无人值守）
- OctoBot/hummingbot：不符（配置型用户/做市方向）

## 2. AI/LLM 层调研结论

**核心事实：LLM 决策不可复现。** TradingAgents 官方 README 承认"同一 ticker+date 跑两次结果可能不同"，且历史社交/新闻数据存在隐性 look-ahead。**不可复现 = 无法回测验证 = 违反本项目的严谨流程原则。**

LLM 的可靠位置（按可靠度降序）：
1. **研报/新闻消化**（最靠谱）：非结构化 → 结构化事实，是压缩不是预测，幻觉可对照原文校验
2. **复盘反思**：决策日志 + 已实现收益 → 教训注入下次分析（TradingAgents 的 reflection 机制，全场最值得抄的设计）
3. **风控审计**：LLM 作为独立"反方"审查即将执行的信号——**只有否决权（VETO），没有开仓权**
4. **情绪因子**（谨慎）：LLM 输出降格为数值因子，走标准因子检验流程
5. ~~信号生成/直接决策~~：**禁止**

项目取舍：

| 类别 | 项目 | 许可证 | 用法 |
|---|---|---|---|
| 抄代码 | TradingAgents (90k★) | Apache-2.0 | 多角色分析编排、decision log/reflection、多 provider 抽象 |
| 抄代码 | Vibe-Trading (17.9k★, 2026新秀) | MIT | Shadow account（影子仓）、mandate gate（授权门控）设计 |
| 抄代码 | qlib (45.7k★) | MIT | PIT 数据库（防 look-ahead）、因子流水线（后期） |
| 抄思想 | ai-hedge-fund (60.9k★) | MIT | agent→risk→PM 分层结构；persona 信号本身无验证价值 |
| 抄思想 | LangAlpha | Apache-2.0 | "持久化研究工作区、论点随新数据迭代"理念 |
| 回避 | FinceptTerminal | AGPL+商业双许可 | 商业条款有法律风险（违者索赔 $50k/年起）、维护降频 |
| 回避 | maybe | AGPL-3.0 | 已归档（2025-07 停止维护） |
| 回避 | FinRL 直接交易 | MIT | RL 回测-实盘鸿沟对个人稳健系统风险过高 |

**架构铁律：LLM 输出永远是"给规则系统的输入"或"对规则系统的否决"，绝不直接成为订单。**

## 3. 收益预期校准（来自 CBOE 官方 39.5 年数据）

给策略设计定锚，防止不切实际的预期：

| 策略 | 长期年化 | 波动率 | 最大回撤 | 来源 |
|---|---|---|---|---|
| S&P 500 买入持有 | 11.1% | 15.2% | -50.9% | CBOE BXM Factsheet 1986-2025 |
| BXM（ATM 备兑） | 8.5% | 10.7% | -35.8% | 同上 |
| PUT（现金担保卖沽指数） | 9.54% | 9.9% | -35.5% | Wilshire/CBOE 32.5年研究 |
| BTC 月度 ATM 备兑 | ~13.9%（仅22个月样本） | — | -26.1% vs HODL -46.6% | CryptoVol 2024-2026 回测 |
| BTC 资金费率套利（当前） | **~2.9%**（2026-07 实测） | 低 | 交易所信用风险 | Binance API 实抓 |

**含义**：
- 期权卖方策略的本质是**卖波动率溢价 + 平滑收益曲线**，不是"免费加收益"；牛市系统性踏空是数学必然
- 任何声称稳定年化 >20% 的"稳健"策略都应默认过拟合或欺诈
- 本系统的合理目标：**年化 8-15%，最大回撤 <20%**，跑赢"拿住不动"的方式是靠回撤控制而非收益率

## 4. 监管环境（2026-07 时点，影响期权线）

- **2026-05-22 八部门方案**：富途/老虎/长桥被立案处罚，两年过渡期内存量大陆客户只能卖出+转出资金，禁止新增买入和入金；期满彻底关闭大陆服务
- **推论**：期权卖方策略（需持续开新仓）在富途/老虎通道已不可执行；可行通道收敛至 IBKR 直开（IBKR 未被点名，但后续是否纳入执行范围无明确结论）
- **加密**：大陆 2021 年起全面禁止加密交易业务；Deribit 强制 KYC 且大陆不在服务地区
- **系统设计影响**：本系统聚焦数字货币线为主战场；期权线降级为"人工 + 工具辅助"，不做自动化（通道风险 > 技术价值）
