# 01 — 要不要自己造？（Build vs Buy 决策论证）

> 写给你本人。这是你在犹豫的那个问题的完整答案。

## 直接结论

**不要从零造，也不要"直接用现成的"——正确答案是第三条路：组装。**

- 用现成的开源框架（freqtrade）做 80% 的地基：下单、交易所对接、回测、基础风控、UI
- 自己（让 AI）只造 20% 的薄壳：你的策略、统一看板、AI 分析层、纪律状态机

## 为什么不从零造

交易引擎是量化系统里 bug 代价最高的部分。断线重连、部分成交、精度舍入、API 限频、时钟漂移——每一个坑在真实市场里都是真金白银。freqtrade 被数千人的实盘资金验证了十年。你从零写一个，等于放弃十年的免费测试。

还有一个更隐蔽的成本：**从零造会让你把精力花在工程上而不是投资上。** 你的目标是复利现金流，不是成为交易系统工程师。工程只是手段。

## 为什么不"直接用现成的"

如果直接用 freqtrade 裸奔，你会缺三样东西，而这三样恰恰对你最重要：

1. **纪律状态机。** 你说你信奉"回测→模拟→小额→大额"，但现成工具不强制这个流程——切到实盘只是改一个配置项，深夜里手痒 30 秒就能跳过所有验证。我们要造的系统把这个流程做成硬性关卡：dry-run 不满 28 天、小额实盘不满 56 天且不足 30 笔交易，系统就不出具升级建议。**这个功能市面上没有现成的，而它是整个系统里对你价值最大的部分。**

2. **AI 分析层。** 研报消化、每笔交易的自动复盘、开仓前的"反方审计"。调研结论很明确：LLM 直接炒币不靠谱（决策不可复现，无法回测），但 LLM 做研究助理和风控反方非常靠谱。这个组合形态市面上也没有现成的。

3. **统一看板。** 你要的"以 UI 形式和用户交互"——把交易、研究、复盘、风控聚合在一处。FreqUI 只有交易那一块。

## 一个必须先接受的数字

CBOE 官方 39.5 年数据：S&P 500 买入持有年化 11.1%；最好的期权现金流指数（PUT）年化 9.54%。BTC 备兑月度策略两年回测年化 ~14%（样本太少，仅供参考）。

**这意味着：任何声称"稳健年化 20%+"的东西，默认是过拟合或骗局。** 你这个系统的合理期望是**年化 8-15%、最大回撤控制在 20% 以内**。它跑赢"拿住不动"的方式不是收益更高，而是回撤更浅、过程可持续、你拿得住。

复利的数学：年化 12%，6 年翻倍，20 年 10 倍。听起来慢，但这是有 40 年数据支撑的"慢"，胜过 90% 的散户实际战绩（你自己的美股经历就是样本）。

## 成本估算（组装路线）

- **开发**：架构文档已完成（本次）。剩余开发量约 4-8 周，大部分任务可由便宜模型按 `docs/system/03-tasks.md` 执行，少数标 🧠 的任务需要强模型。
- **运行**：VPS ~$10-20/月 + LLM API ~$10-30/月（研究/复盘用便宜档模型，用量可控）
- **学费**：小额实盘阶段按总资金 5% 投入，把它心理上计为"学费预算"。

## 最大的风险不是技术

按优先级排序，这个项目真正的风险是：

1. **你自己跳过流程**（手痒、FOMO、回本心切）——系统的核心功能就是防这个
2. **过拟合的策略**（回测年化 50% 的策略实盘亏钱）——walk-forward + Monte Carlo 防这个
3. **交易所风险**（FTX 式暴雷）——只用头部交易所、API key 禁提现、大部分资产冷存储，交易账户只放策略需要的钱
4. 技术 bug——排最后，因为地基是别人验证过的

## 判断这件事成功的标准

12 个月后回看，成功不是"赚了多少"，而是：
- [ ] 系统 7x24 稳定运行，你每天只花 <10 分钟看 Telegram
- [ ] 每一笔交易都有记录、有复盘、可解释
- [ ] 你没有一次绕过状态机手动干预（或干预过但被系统记录并复盘了）
- [ ] 实盘表现与回测偏差在可解释范围内
- 收益为正是这四条的自然结果，而不是目标本身
